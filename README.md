## 프로젝트 소개

> LinkShop은 사용자가 자신만의 쇼핑몰 링크를 생성하고, 대표 상품을 등록 및 공유할 수 있는 서비스입니다.
> 
> 
> 사용자는 링크샵을 등록하고, 다른 사람의 링크샵을 둘러보며 좋아요 및 공유가 가능합니다.
> 
> 작지만 소중한 프로젝트를 목표로, 모든 팀원이 직접 손발 맞춰 하나하나 구현하며 완성했습니다.
> 
> 처음부터 끝까지 고민하고, 열심히 만든 결과물을 소개합니다.
>

## 주요 기능

- 링크샵 생성 및 수정기능
- 상품 이미지 업로드 및 관리
- 링크샵 검색 및 정렬 필터, 무한스크롤
- 링크샵 상품 목록, 공유 & 좋아요
- 반응형 웹 지원 (PC/태블릿/모바일)

## **2. 채택한 개발 기술과 브랜치 전략**

- **React**
    - 컴포넌트화를 통해 추후 유지보수와 재사용성을 고려했습니다.
    - 링크샵 상품 목록 등 중복되어 사용되는 부분이 많아 컴포넌트화를 통해 리소스 절약이 가능했습니다.
    
- **eslint, prettier**
    - 정해진 규칙에 따라 자동적으로 코드 스타일을 정리해 코드의 일관성을 유지하고자 했습니다.
    - 코드 품질 관리는 eslint에, 코드 포맷팅은 prettier에 일임해 사용했습니다.
    
- **브랜치 전략**
    - Git-flow 전략을 기반으로 main, develop 브랜치와 feature 보조 브랜치를 운용했습니다.
    - main, develop, Feat 브랜치로 나누어 개발을 하였습니다.
        - **main** 브랜치는 배포 단계에서만 사용하는 브랜치입니다.
        - **develop** 브랜치는 개발 단계에서 git-flow의 master 역할을 하는 브랜치입니다.
        - **Feat** 브랜치는 기능 단위로 독립적인 개발 환경을 위하여 사용하고 merge 후 각 브랜치를 삭제해주었습니다.

│   └── Toast/                # 등록 완료

│   ├── LinkDetailPage/      # 상세 페이지
│   ├── LinkEditPage/        # 수정 페이지
│   ├── LinkPostPage/        # 생성 페이지
│   └── ListPage/            
├── styles/                 
│   └── AppMain.module.css
├── App.jsx                  # 앱 최상위 컴포넌트 (라우터 포함)
├── AppMain.jsx              # 메인 페이지용 별도 컴포넌트
└── main.jsx                 # 앱 진입점

## 7. 프로젝트 후기

**🍊 문수현**

이번 프로젝트를 통해 프론트엔드에 대해 배운 지식을 실제로 적용해보며 프로젝트를 완성해볼 수 있었던 점이 좋은 경험이었습니다. 다양한 배경과 성향을 가진 팀원들과 함께 협업하면서 새로운 시각을 얻을 수 있었던 점도 인상 깊었고, 서로의 생각을 존중하며 소통하는 과정이 의미 있었습니다. 특히, 각자의 역할을 충실히 수행하면서도 서로 피드백을 주고받으며 한 방향으로 나아가는 팀워크가 인상 깊었고, 작은 기능 하나를 구현하더라도 사전에 충분한 논의와 설계가 왜 중요한지 몸소 느낄 수 있었습니다. 이번 프로젝트를 함께 하며 겪은 시행착오와 그 속에서 얻은 배움들이 앞으로의 개발 여정에 큰 자산이 될 것이라 믿습니다. 바쁘신 와중에도 끝까지 함께해주신 팀원분들 덕분에 많은 배움과 소중한 기회를 얻어갑니다. 모두 고생 많으셨고, 앞으로도 좋은 일들만 가득하시길 바랍니다!

**👻 김진형**

이번 프로젝트를 통해 정말 많은 것을 배울 수 있어 감사했습니다. 서로 다른 강점을 가진 팀원들과 함께 하면서 저 또한 성장할 수 있었습니다. 각자의 자리에서 최선을 다해준 모든 팀원들에게 깊이 감사드립니다. 때로는 쉽지 않은 순간들도 있었지만, 함께 이겨낸 시간이 더욱 소중하게 느껴집니다. 서로 존중하고 배려하며 협력했던 과정이 큰 의미로 남습니다. 짧은 기간 동안 놀라운 결과를 만들어낸 모든 분들의 노고에 진심으로 박수를 보냅니다.  모두 수고 많으셨습니다. 앞으로도 건승하시길 바랍니다.

**😎 유서현**

정규 수업에서 간단하게만 실습을 해보다가, 팀 프로젝트를 진행하게 되면서 배웠던 내용들을 최대한 끄집어내어 활용하려는 과정이 어렵기도 했지만 어떤 부분이 부족한지 알 수 있는 좋은 기회였습니다. 2주의 기간동안 구현을 최우선 목적으로 두고 진행하다가 멘토님께 피드백을 받으며 초반 체계적인 설계의 중요성을 느끼기도 했습니다. 빠르게 작업을 진행하려다 보니 트러블슈팅 내용을 기록하지 못한 점은 아쉽지만 프로젝트를 잘 완주하게 되어 뜻깊게 느껴지고 팀원분들께 감사합니다.

**🐬 조진우**

개발 공부를 시작하면서 프로젝트를 해본다는 것이 막연하게 만 느껴졌지만, 프로젝트를 직접 팀원들과 해보면서, 협업에서 어떤 것이 중요하고 팀원들과의 소통이 얼마나 중요한지 정말 잘 느끼게 된 계기가 되었습니다. 기초 프로젝트이기 때문에 간단할 줄만 알았지만, 작은 기능에도 얼마나 많은 시간과 작업이 들어가는지 알게 되었고, 그 가운데 얼마나 많은 소통을 해야 하는지도 알게 된 계기가 되었습니다. 또한 지금까지 개발 공부를 하면서 이해가 되지 않는 부분들도 있고 어려워서 스트레스를 받았던 부분들도 있었는데, 직접 코드를 작성해보고 팀원들이 작성한 코드를 보고, 서로 서로 소통하다 보니 전부는 아니지만 지금까지 배웠던 많은 양의 지식들 중 일부들이 머리 속에 정리가 되면서 이해가 되는 정말 좋은 시간이였던 것 같고, 오랜 기간은 아니였지만, 열심히 고생하고 노력해준 팀원 분들에게 감사하고 고생해셨다는 말을 전합니다. 

👑 양지선

이번 프로젝트는 저에게 많은 의미를 남긴 소중한 시간이었습니다. 처음 하는 프로젝트여서 초반에 어려운 것들이 많았지만, 끝까지 팀원분들과 함께 할 수 있어서 뿌듯했습니다. 특히 개발뿐만 아니라 문제를 해결하는 태도, 소통하는 방법, 그리고 하나의 결과물을 함께 만들어가는 즐거움을 배웠습니다. 중간에 힘들고 막막했던 순간도 있었지만, 함께했던 팀원들 덕분에 끝까지 포기하지 않고 달릴 수 있었습니다. 모든 걸 하나하나 만들어 가는 과정에서 팀원 분들이 안 계셨으면 너무 막막했을 것 같아요. 협업을 통한 경험을 해 보면서 이슈가 생기면 함께 해결하는 과정이 가장 의미가 깊었던 것 같습니다. 프로젝트 진행과정에서 서로 믿고 소통하면서 많이 배웠습니다. 깃허브을 통해 기록한 과정들이 미흡하지만 기억에 오래 남을 것 같아요:) 그리고 친절하게 방향을 잡아주고, 응원해주셨던 **멘토님** 덕분에 더 원활하게 끝까지 올 수 있었던 것 같습니다. 팀원들 모두 고생했다고 전하고 싶습니다🧡
